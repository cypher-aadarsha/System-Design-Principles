What is an Operating System?

An Operating System (OS) is a software program that acts as an intermediary between a computer user and the computer hardware.

Primary Goals

  • Execute user programs and make solving user problems easier.
  • Make the computer system convenient to use.
  • Utilize the computer hardware in an efficient manner.

Key Analogy

Think of an OS as the government of a country. It doesn't do "real" work on its own, but it provides the infrastructure and environment for other programs (citizens) to do useful work.

Two Views of an OS: The User View

The user's view of the computer varies according to the interface being used.

PC / Workstation

Users expect ease of use and good performance. The OS is designed for a single user, with a focus on responsiveness and user experience (e.g., Windows, macOS).

Mainframe / Minicomputer

Users (terminals) share resources. The OS is designed to maximize resource utilization, ensuring all users get fair access (e.g., z/OS).

Handheld / Mobile

Users expect a simple interface, long battery life, and connectivity. The OS is optimized for touch input and power efficiency (e.g., Android, iOS).

Two Views of an OS: The System View

From the computer's point of view, the OS is the program most intimately involved with the hardware. In this context, the OS is a resource allocator.

  • Manages all hardware resources (CPU time, memory space, file storage, I/O devices).
  • Decides between conflicting requests for efficient and fair resource use.
  • Acts as a control program to manage the execution of user programs to prevent errors and improper use of the computer.

Evolution of Operating Systems

The Early Days: Serial Processing (1940s-1950s)

  • No OS. Programmers interacted directly with the hardware.
  • Computers ran one job at a time, entered via punch cards or paper tape.
  • Extremely inefficient due to setup time between jobs. The CPU was idle most of the time.

Evolution: Batch Systems (1950s-1960s)

The first rudimentary OS was born to manage batches of jobs.

  • An operator would group similar jobs (e.g., all FORTRAN programs) together into a "batch".
  • The OS, a simple program called a "monitor," would load a job, run it, and then automatically load the next one.
  • This improved performance by reducing setup time, but the CPU was still often idle (e.g., waiting for slow I/O).

Evolution: Multiprogramming (1960s-1970s)

A major leap in efficiency by keeping the CPU busy.

  • The OS keeps several jobs in memory simultaneously.
  • When one job has to wait for an I/O operation to complete, the OS switches the CPU to another job.
  • This significantly increased CPU utilization and system throughput.
  • Requires memory management and CPU scheduling from the OS.
Designed by Nikesh Shah & Niuj Poudel

Evolution: Time-Sharing (Multitasking)

A logical extension of multiprogramming, allowing interactive use.

  • The CPU switches between jobs so frequently that users can interact with each program while it is running.
  • This creates the illusion that each user has their own dedicated machine.
  • Requires sophisticated CPU scheduling algorithms and memory protection.
  • Led to the development of interactive systems and personal computing.

Evolution: Personal & Modern Systems

Personal Computers (PCs)

Appeared in the 1970s. Early OSs (like MS-DOS) were not multi-user or multitasking. Evolved into powerful systems (Windows, macOS, Linux) with features from earlier, larger systems.

Modern Era

Focus on distributed systems (networked computers), mobile devices (Android, iOS), and virtualization (running multiple OSs on one machine). Security and user interface are paramount.

Types of OS - In Detail

Distributed OS

Manages a group of independent computers and makes them appear to be a single computer. Users are unaware of where their programs are being run or where their files are stored.

Real-Time OS (RTOS)

Designed for systems where the time taken to process an input is a critical constraint. Used in industrial control, robotics, and scientific instruments. (e.g., VxWorks, QNX).

Mobile OS

Designed specifically for smartphones, tablets, and other handheld devices. Key features are touch interface, power management, and connectivity. (e.g., Android, iOS).
